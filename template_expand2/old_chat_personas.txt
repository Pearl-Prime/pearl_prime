# Location Overlay Pack — Template, Example, and Review Rules
## 1) **What they are building (one sentence)**
Location overlays are **environment-only YAML files** that inject physical, cultural, and situational context into stories — **without** adding emotion, plot, or meaning.
These overlays make **persona + engine + location** work automatically.
---
## 2) **Canonical template (this is the contract)**
yaml
# locations/<city>.yaml
# ENVIRONMENT-ONLY LOCATION OVERLAY
# No emotion. No plot. No therapy language.
environment:
  places:
    - <physical place RN might be>
    - <another physical place>
  sensory:
    - <neutral sensory detail>
    - <another sensory detail>
  rhythms:
    - <time-based or routine pattern>
    - <shift / commute / seasonal rhythm>
  cultural_notes:
    - <matter-of-fact social norm>
    - <local behavioral tendency>
  climate:
    - <weather / seasonal condition>
  infrastructure:
    - <transport / building / systems fact>

### Hard rules (non-negotiable)
* ✅ nouns > adjectives
* ✅ observable facts only
* ❌ no feelings (e.g. *lonely, anxious, heavy*)
* ❌ no story verbs (*wonders, hopes, remembers*)
* ❌ no meaning (*represents, symbolizes*)
If it reads like a novel sentence, it’s wrong.
---
## 3) **Gold-standard example (Chicago, healthcare RN)**
yaml

environment:
  places:
    - hospital corridors
    - break room with vending machines
    - L-train platform
    - parking garage stairwell

    - rubber soles on tile
  rhythms:
    - night shift handoff
    - early morning commute before sunrise
    - weekend double shifts
  cultural_notes:
    - practical humor
    - direct communication
    - reserved with strangers
  climate:
    - long winters
    - sharp temperature swings
  infrastructure:
    - elevated train lines
    - large public hospitals

Why this passes:
* Everything is **placeable**
* Nothing tells a story
* Nothing implies emotion
* Engines stay in control
This is the standard.

## 4) **How review + greenlight works (fast, mechanical)**
You only review **one file per city family**.
### Review checklist (takes ~2 minutes)

   * If no → rewrite
3. **Any emotional adjectives?**
   * If yes → delete
4. **Any verbs describing inner state or meaning?**
   * If yes → delete
If it passes → **greenlight all remaining overlays from that contributor**
No need to line-edit every file.

Once approved:
* Overlay is registered in locations/registry.yaml
* Personas reference it via locale_key
* Generator auto-applies it

* Engines remain untouched
Their work scales across **every topic and engine**.
