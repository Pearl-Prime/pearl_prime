# Ownership Matrix

**Purpose:** One place to see which repo and which surfaces own GitHub governance,
automation, runtime, and operational workflows. This reduces drift and duplicate
ownership assumptions.

## Repo Roles

| Repo | Role | Production schedules? |
|------|------|------------------------|
| `phoenix_omega_v4.8` | Primary production repo for Phoenix Omega system code, governance, workflows, catalog, manga, Pearl News, EI, video, and operator docs | Yes |
| `Qwen-Agent` | Adjacent runtime / backup / experiment repo; may still contain helper runtime material but is not the default merge target for Phoenix feature work | Limited / separate governance |

## Ownership In This Repo

| Path / domain | Owner | Notes |
|---------------|-------|-------|
| `.github/workflows/*` in this repo | `phoenix_omega_v4.8` | Canonical GitHub workflow definitions for this repo |
| `config/governance/*` | `phoenix_omega_v4.8` | Machine-readable governance and required-check policy |
| `docs/GITHUB_*`, `docs/BRANCH_PROTECTION_REQUIREMENTS.md`, `docs/RELEASE_POLICY.md` | `phoenix_omega_v4.8` | Human-readable GitHub governance authority docs |
| `scripts/ci/verify_github_governance.py` and related governance validators | `phoenix_omega_v4.8` | CI-facing governance enforcement |
| `scripts/audit/*` that write repo audit artifacts | `phoenix_omega_v4.8` | Audit/reporting utilities for repo governance and operational review |
| `artifacts/audit/*` | generated by repo workflows/scripts | Evidence output, not hand-edited source |

## Canonical Rule

If a file is governed by this matrix:

1. update the canonical file in this repo
2. update docs and config in the same PR when behavior changes
3. do not treat stale copies in old branches as merge vehicles

## Drift Rule

When ownership is unclear:

1. trust current `origin/main`
2. verify from git history and workflow reality
3. document the decision in `docs/PEARL_GITHUB_STATE.md` or a dedicated harvest plan
