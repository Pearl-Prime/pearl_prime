# Manga Production-Line Agents

## Operating principle

The production line is a chain of explicit contracts, not a collection of personas.
Each role receives named artifacts, emits named artifacts, runs acceptance tests, and
hands failures to a specific owner.

The machine-readable authority is:

`config/manga/production_line_agents.yaml`

## Required artifact chain

`series_brief → story_plan → panel_plan → layer_plan → structural_plan → assembly_manifest → lettering_plan → page/webtoon_plan → proof_packet → blind_read_scorecard → release_decision`

Every artifact must preserve:

- series_id
- episode_id
- chapter_id
- panel_id
- beat_id
- doctrine_id
- layer_role
- support_zone_id

## Roles

### manga_editor

Owns audience, genre promise, doctrine subtlety, episode scope, and reader bar.
The editor does not approve technical gates by interpretation.

### manga_writer

Authors visual action, dialogue, silence, choices, escalation, metaphor, analogy,
parable, and closing propulsion. The writer does not paste self-help prose into bubbles.

### manga_story_doctor

Repairs story causality, emotional escalation, anti-lecture failures, and weak mode
integration. Every repair must retain trace IDs.

### manga_panel_director

Maps each beat to panel purpose, camera, structural template, and page rhythm.
A beautiful panel that does not carry its beat is a failed panel.

### manga_layer_art_director

Owns raw layer truth, selected-component cleanup, support zones, composition sidecars,
and structural assembly. `layer_00.png` is never assembly authority.

### manga_letterer

Owns locale, bubble readability, SFX, vertical Japanese, and furigana. Fallbacks must
be explicit and may block release when the locale contract requires the missing mode.

### manga_layout_artist

Owns page and webtoon reading order, crops, gutters, panel hierarchy, and export
readability.

### manga_qa_reader

Runs automated gates and the human blind-read process. Human approval cannot be
generated by code.

### manga_pm

Tracks branches, PRs, SHAs, proofs, blockers, operator approvals, release readiness,
and the final verdict.

## Failure handoffs

Failures move upstream to the role able to correct the source:

- story/doctrine → writer or story doctor
- panel purpose/template → panel director
- layer purity/support → layer art director
- lettering/locale → letterer
- page reading order → layout artist
- reader rejection → editor
- missing proof/approval/merge → PM

No downstream role should silently repair an upstream contract and erase the trace.
