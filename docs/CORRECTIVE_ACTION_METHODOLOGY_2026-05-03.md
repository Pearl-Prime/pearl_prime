## §1 What this PR ships

### §1.1 `config/manga/corrective_prompts_for_dashboard_22.yaml` (1 file, 22 entries)

Generated by `scripts/manga/build_corrective_prompts.py`. For each of the 22 stillness_press dashboard series, contains:

- **series metadata** — series_id, genre_family, locale, brand_id, diagnosis_class (from PR #838's dashboard 22-failure diagnosis)
- **filled `character_design`** — all 9 lockout axes + sub-attributes per PR #842's 12-axis vocabulary
- **individuation constraint check** — collision count vs other 21 series; pass/fail status; auto-rotated axis values where original draft collided
- **corrective prompt** — Animagine XL 4.0 token order per PR #842's algorithm §1.2; sampler/scheduler/steps/cfg per PR #807's schnell-fix workflow
- **deterministic seed** — `sha256(series_id + ":corrective_v2") mod 2^31` for reproducibility
- **expected_drift_reduction** — heuristic per diagnosis class

**Validation result on the 22:** constraint_solver_pass: **True**, individuation_collisions_detected: **0**. The auto-rotation fallback (per PR #842 algorithm §1.4) was triggered once on `stillness_press_pressure_map` (rotated `nose_construction.bridge`, `accessories.signature_item`, `mouth_jaw.lip_shape`, `palette.accent`) to resolve a 6-of-9 collision against `stillness_jp_01`'s manager-archetype profile.

### §1.2 QA harness (3 scripts shipped — minimal commercial-clean implementations)

The brief expected these to ship with cookbook v2 (#831). They didn't. This PR ships minimal versions:

- `scripts/image_generation/qa/manga_register_check.py` — drift score per image. Reads `F_forbidden_drift_patterns` per genre from `drawing_tradition_per_genre.yaml`; computes drift_score against rendered output's prompt + (deferred) Florence-2 reverse-prompt caption.
- `scripts/image_generation/qa/genre_drift_detector.py` — per-image expected-token match. Reads `H_token_mapping.animagine_xl_4_0.positive` per genre; computes match_ratio against the rendered prompt.
- `scripts/manga/individuation_pairwise.py` — pairwise face-distance matrix. Implements PR #842's primary metric (DeepFace + FaceNet-512 cosine) with CLIP ViT-B/32 fallback. Thresholds from `character_individuation_metric.yaml`.

**Caveat:** Florence-2 / WD-EVA02-Tagger reverse-prompt is DEFERRED (not pre-installed). Without it, `manga_register_check.py` validates prompt-construction discipline only, not rendered pixel content. Once Florence-2 is integrated (~30-60 min engineering follow-up), drift_score reflects actual pixel-vs-genre alignment. **This is documented as a known gap, not a blocker.**

### §1.3 Build script

`scripts/manga/build_corrective_prompts.py` — regeneratable. Operator can re-run any time inputs change (cookbook update, drawing-tradition update, axes update). Outputs the YAML deterministically.

**Threshold calibration (Phase 4 of this methodology):**
The thresholds from `character_individuation_metric.yaml` (`< 0.25` reject, `[0.25, 0.40)` borderline, `≥ 0.40` pass) are PROPOSED. Empirical calibration on the 22 dashboard PNGs is required:

1. Run `individuation_pairwise.py` on the **ORIGINAL** 22 PNGs (operator-eyeballed look-alike). Most pairs should cluster well below 0.25 if the metric matches operator perception.
2. If FaceNet-512 returns "all distinct" (>0.40) on the original 22 despite operator-eyeballed look-alike: fall back to CLIP ViT-B/32 image-image cosine.
3. Use the calibrated 22-image distribution to set production threshold at `1.5 × median pairwise distance` from a held-out 5-character "obviously distinct" set.

## §3 Phase 2 application guide (post-PR-C smoke pass)

Once the 22 smoke test passes, the methodology scales to ~110 new series across 11 brands per the production campaign brief. Confidence-tiered sampling:

**Auto-promote tier (high confidence):**
- Diagnosis class = `needs_workflow_fix` AND drift_score < 0.1 AND individuation_distance ≥ 0.45
- No operator review needed; promote v2 → canonical

**Operator-sample tier (10% per brand):**
- Random 10% of each brand's series
- Operator visual review of side-by-sides
- Calibrates whether auto-promote rule is producing visible regressions

**Operator-borderline tier:**
- Diagnosis class ∈ {`needs_LoRA_plus_multiple`, `needs_character_individuation_pipeline`}
- Drift score in 0.1-0.3 OR individuation distance in 0.25-0.40
- Operator visual review of every borderline series

**Hard-stop tier:**
- 22 individuation collisions on first solver run despite full 9-axis fill — likely indicates catalog axis-vocabulary saturation; expand vocabulary or accept limited near-twin pairs
