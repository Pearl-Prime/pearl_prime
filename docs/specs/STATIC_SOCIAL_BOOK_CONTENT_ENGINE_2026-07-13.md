# Static Social Book Content Engine

**Date:** 2026-07-13  
**Scope:** Organic static social content generated from Pearl Prime books, freebies, and enhancement artifacts.  
**Inputs:** `docs/book_deep_research.txt`, attached static self-help strategy brief, live GHL E1-E9 Waystream workflow.  
**Status:** Operating spec / implementation target. This replaces quote-card-first assumptions in older 48 Social docs.

## Executive Doctrine

The strongest static self-help content is not a feed full of generic quote cards.

The winning unit behaves like a miniature self-help chapter:

1. Recognize the reader.
2. Validate the experience without making them powerless.
3. Explain one mechanism.
4. Offer one reframe.
5. Give one micro-practice.
6. Add one troubleshooting note when useful.
7. End with a natural invitation.

Static posts should make the reader feel seen, give them one usable idea, and give the platform a reason to save, share, complete, reply, or click.

## What Changes

Older 48 Social language over-weighted quote cards. That is no longer the production target.

Default static mix per 20 non-video posts:

| Class | Target | Function |
| --- | ---: | --- |
| Practical teaching posts | 8 | Saveable utility and book authority |
| Recognition / validation posts | 5 | Emotional specificity and shareability |
| Personal or reader-style stories | 3 | Trust, relatability, proof of method |
| Strong opinions / reframes | 2 | Conversation and thought leadership |
| Original quote with commentary | 1 | Shareable identity signal, not filler |
| Offer / invitation | 1 | Freebie, book, bundle, or list growth |

Quote-style posts must stay below 10-15% of the static feed unless a brand-specific test proves otherwise. Borrowed famous quotes are disallowed as default content. Original lines from the book are allowed only when paired with commentary, mechanism, or practice.

## Book-To-Social Extraction

For each rendered book, the content engine should extract these source surfaces:

| Book surface | Social use |
| --- | --- |
| Chapter clear point | Carousel hook, LinkedIn PDF thesis, X/Threads claim |
| Mechanism explanation | Teaching slide, diagram, caption proof |
| Micro-practice / exercise | Saveable carousel, Pinterest Pin, Facebook practical post |
| Troubleshooting beat | Slide 7-8 obstacle card, email reassurance, reply prompt |
| EXTERNAL_STORY | Personal narrative carousel, Facebook story, TikTok photo sequence |
| AUTHOR_DISCLOSURE / AUTHOR_COMMENTARY | Trust-building photo caption or Threads post |
| ANALOGY / METAPHOR | Simple diagram, motif graphic, single-line reframe |
| CALLBACK_RETURN | Cohesion thread, "remember this" follow-up post |
| CITED_EVIDENCE | LinkedIn authority post, diagram, careful caption note |
| Memorable line | Original quote card with expanded caption |

The social engine must not invent proof, stories, citations, author disclosure, or therapeutic claims that are not present in the book, atom bank, or approved campaign plan.

## Required Post Families

For a full book package, generate 30-50 static-social assets across platforms. The exact count may vary by book length and market, but the package should include:

| Asset family | Target per book | Notes |
| --- | ---: | --- |
| Instagram carousels | 5-8 | Primary static format for depth, saves, shares |
| LinkedIn PDF/document posts | 2-4 | Work-relevant framing; strongest for professional personas |
| Pinterest vertical Pins | 6-10 | Searchable, evergreen, practical assets |
| Facebook photo/text posts | 4-6 | Relatable stories, shareable validation, discussion |
| Threads/X text posts | 6-10 | Native conversational ideas, claims, and prompts |
| TikTok photo-mode sequences | 2-4 | Younger audience discovery; diary/list style |
| Original quote-with-commentary cards | 1-3 | Must include original interpretation or practice |
| Offer/invitation posts | 2-4 | Freebie, book, bundle, companion workbook |

Do not produce the same graphic everywhere. One book idea is adapted by platform behavior, not copied across channels.

## Universal Writing Arc

Use this default arc for educational content:

1. Hook: a specific private behavior, contradiction, distinction, or compassionate challenge.
2. Recognition: "This is the thing you have been doing."
3. Validation: reduce shame, avoid permanent-damage framing.
4. Mechanism: one clear reason the pattern happens.
5. Reframe: the shareable sentence.
6. Practice: one action that can be tried today.
7. Troubleshooting: what to do when the reader resists or the practice feels too small.
8. Invitation: save, reply, share, click, or reflect.

Forbidden hook style:

| Bad pattern | Safer rewrite |
| --- | --- |
| "If you do X, you have trauma" | "If your body reacts to stress by X..." |
| "Your nervous system is broken" | "Your system may have adapted to chronic stress by..." |
| "This one trick fixes everything" | "One small practice that can make the next step easier" |
| "Comment YES!!!" | "What is one promise you could make smaller today?" |

## Instagram Carousel Standard

Default canvas: 1080x1350, 4:5.  
Default length: 6-10 slides.  
Default text load: one idea per slide; 15-45 words per teaching slide.

Nine-slide default:

1. Specific hook.
2. Recognizable situation.
3. Validation / normalization.
4. Mechanism explanation.
5. Key reframe.
6. Practical exercise.
7. Likely obstacle.
8. Closing takeaway.
9. Save / share / reflection invitation.

When using a book enhancement:

| Enhancement | Carousel placement |
| --- | --- |
| EXTERNAL_STORY | Slides 2-4 or a standalone story carousel |
| ANALOGY / METAPHOR | Slide 4 mechanism or slide 5 reframe |
| TROUBLESHOOTING | Slide 7 |
| CITED_EVIDENCE | Slide 4, never as a cold opener for high-validation chapters |
| CALLBACK_RETURN | Slide 8 or caption callback |
| AUTHOR_DISCLOSURE | Caption or photo-led carousel, not a generic quote card |

| Platform | Primary static format | Native behavior | CTA style |
| --- | --- | --- | --- |

| Facebook | Photo plus story, text reflection, albums | Community sharing and relatability | Direct link or discussion prompt |

| LinkedIn | Native PDF/document carousel | Dwell time, professional authority | Comment keyword or first-comment link |

| TikTok | Photo-mode carousel | Identity discovery, diary/list style | Comment/share prompt |
| Bluesky | Text or raw photo | Peer conversation | Open-ended inquiry |
| YouTube Community | Poll, image, text | Re-engage subscribers | Poll vote or simple prompt |

## Visual Direction

Use beauty to stop the scroll, specificity to create recognition, explanation to create trust, and practice to create value.

Preferred visuals:

| Visual | Best use |
| --- | --- |
| Authentic human portraits | Trust, author voice, personal stories |
| Everyday embodied moments | Anxiety, sleep, boundaries, burnout, grief |
| Symbolic objects | Clear metaphor tied to the caption |
| Worksheets and checklists | Practical utility and saves |
| Handwritten notes / journals | Intimacy and reflection |
| Simple diagrams | Mechanism and authority |
| Custom illustration | Gentle conceptual education |

Avoid:

| Avoid | Reason |
| --- | --- |
| Generic meditation stock photos | Low trust, saturated visual language |
| AI-perfect human faces | Undermines wellness trust |
| Borrowed famous quotes with no commentary | Low brand equity |
| Entire captions trapped inside graphics | Hard to read and low completion |
| Constant beige wellness cards | Saturated and forgettable |
| Text over busy faces | Accessibility and comprehension failure |

## Color And Typography

Use brand palettes, but map them into one of these visual systems when a brand has no stronger local style:

| Mode | Palette direction | Best topics |
| --- | --- | --- |
| Somatic healing | deep pine, terracotta, oatmeal, charcoal | anxiety, trauma recovery, body scans |
| Strategic rational | obsidian blue, ochre, bone white, aqua | productivity, habits, leadership |
| Spiritual integration | aubergine, sand, charcoal, lavender | grief, meaning, shadow work |
| Warm humanist | cream, coral, deep teal, warm black | confidence, self-worth, relationships |

Typography rules:

| Rule | Requirement |
| --- | --- |
| Typeface count | No more than two principal typefaces |
| Cover copy | 6-16 words |
| Teaching slide copy | 15-45 words |
| Paragraph alignment | Left-align body copy over three lines |
| Contrast | Must be readable on mobile |
| Brand mark | Present but subordinate to the teaching |

## Safety And Ethics Gate

Every generated static asset must pass these checks:

1. No unsupported diagnosis.
2. No panic bait.
3. No shame-based sales pressure.
4. No claim that the brand replaces therapy, diagnosis, or medical care.
5. No invented citations, testimonials, or reader stories.
6. No clinical term used as a casual insult.
7. No call to action that manipulates vulnerability.
8. No advice that asks the reader to ignore danger, abuse, or medical needs.

## QA Gates

Minimum gates before scheduling:

| Gate | Pass condition |
| --- | --- |
| Platform fit | Asset follows platform-native format and CTA behavior |
| Source trace | Every story, citation, quote, and practice has a source |
| Visual readability | Mobile-safe contrast, size, and hierarchy |
| Specificity | Hook contains a concrete behavior or tension |
| Utility | At least one saveable/practical idea in teaching posts |
| Safety | No diagnosis creep, panic bait, or therapy replacement |
| Density | Quote cards <= 15% of package |
| Mix | Practical + recognition + story classes dominate |

## GHL / Freebie Relationship

Static social assets send users to freebie pages. The GHL workflow then handles the contact-specific evergreen sequence:

1. E1: free tool delivery.
2. E2: second practice.
3. E3: transformation story.
4. E4: book recommendation.
5. E5: more books list.
6. E6: book two recommendation.
7. E7: book three / bundle path.
8. E8: last chance.
9. E9: still here / original free tool reminder.

Social content should not attempt to run the entire nurture sequence inside a post. Its job is to create recognition, trust, and a clear reason to open the free tool.
