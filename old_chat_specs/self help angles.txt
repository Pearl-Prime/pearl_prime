## Terminology

- **Hybrid path**: The production architecture where outline-first planning provides metadata only, and CSV rendering generates prose. Implemented as PhoenixRaisingCreator.build_book(...).
- **CSV bypass path (DEPRECATED)**: Previously _build_outline_first_book, which generated full prose via ctok orchestrator and skipped CSV rendering. This path is now deprecated and raises NotImplementedError.
- **CSV renderer**: The standard Phoenix Raising chapter body pipeline that loads CSV sections, selects a style for each section, hydrates slots, and injects stories/exercises.
- **Opinion mode**: A writing mode toggle (opinion_mode) that selects the opinion CSV bank.
- **Book-frame injections**: Deterministic prose tokens (topic struggle, effort, body signatures, chapter arc openings, progress acknowledgments, forward momentum) injected into chapters via CSV templates.

## High-Level Architecture (Target / Production)

### Step 1 — Outline-First Planning (metadata only)
The system uses outline-first planning to determine per-chapter constraints and intent:

- scenario_condition_id (per chapter)
- triad_spec_id
- focus
- verb
- exercise_family

This planning data is then merged into the book plan used by the CSV renderer.

### Step 2 — CSV Rendering (prose container)
The chapter body prose is rendered from the somatic style-pack CSVs:

- Load all styles for the target chapter
- Choose a style per section (mixed styles per chapter)
- Concatenate the selected sections into a prose container
- Hydrate variable slots using the resolved slot_values (persona/topic/plan/exercises/stories)

### Step 3 — Story / Exercise slotting (beat-map aware)
Stories and exercises are injected using the beat-map assembly system. The CSV prose remains the “container”, with stories/exercises slotted into the appropriate beats/sections.

## LLM Spine v0.1 Implemented (LM Studio) — 2026-01-08

This repo now contains a single shared “LLM spine” for LM Studio-backed LLM usage. It standardizes transport, retries, chunking, prompt contracts, safety checks, and structured results.

### Goals

- One API surface: run_task(task_spec: TaskSpec, task_input: TaskInput, ...) -> TaskResult
- LM Studio local backend at http://127.0.0.1:1234/v1
- Target model: qwen2.5-14b-instruct
- Supported languages:
  - en
  - zh-cn
  - zh-tw (Traditional-only enforced)
  - ja (lightly casual タメ口 + occasional slang)

### Authoritative code locations

- Package: llm_spine/
  - tasks.py — task contracts (TaskSpec, TaskInput, TaskResult, ChunkResult)
  - prompts.py — prompt registry + rendering + prompt_fingerprint
  - chunking.py — language-aware chunking + stable chunk_hash
  - safety.py — deterministic pre/post checks + zh-tw/ja post-gates
  - client.py — LM Studio HTTP transport, preflight, circuit breaker, concurrency control
  - runner.py — entrypoint run_task(...), retries, JSON parsing, truncation handling
  - probe.py — LM Studio integration probe (multilingual)
  - cache.py — optional dev cache (fingerprint-keyed)

### Single entrypoint

- from llm_spine import run_task, TaskSpec, TaskInput

The spine is intended to be the *only* place that talks to POST /v1/chat/completions.

### Prompt contracts (traceability)

Each call records:

- prompt_template_id
- prompt_version
- prompt_fingerprint (sha256 of rendered prompt)

### Hierarchical IDs (v0.1.1 addendum)

- run_id — supplied by caller (Phoenix Raising run id / book id / etc.)
- task_id — stable hash of (run_id + task_name + language + model + prompt_fingerprint + metadata)
- chunk_id — stable hash of (task_id + chunk_index + chunk_hash)

### Reliability mechanisms

- **Preflight**: GET /models (+ optional ping completion) via LmStudioClient.preflight()
- **Truncation handling**: if finish_reason == "length" → parse_error="truncated_output_finish_reason_length"
- **Retries**: configurable via RetryPolicy:
  - on HTTP errors
  - on empty content
  - on truncation
  - on schema invalid
- **Circuit breaker**: trips after N consecutive failures, holds open for cooldown
- **Concurrency control**: semaphore limits in-flight requests
- **Stall detector (v0.1.1)**: optional wall-clock deadline (TimeoutPolicy.wall_clock_deadline_s) returns timeout_wall_clock_deadline

### Response modes

- text
- json_strict
  - schema-first prompt wording enforced
  - JSON parsing with balanced {...} fallback
  - schema validator is applied when provided
  - hard fail on truncation or schema invalid
- json_loose (best-effort JSON extraction)

### Deterministic safety layer

- **Pre-check** (deterministic keyword gates): blocks/warns on disallowed requests
- **Post-check**: scans generated content for disallowed patterns (e.g., clinical “cure/guarantee/clinically proven” claims)
- Enforcement by safety_profile:
  - strict: block
  - standard: warn (attach evidence)
  - dev_repair_ok: allow repair tasks to run and re-check

### Language post-checks + deterministic repair behavior (v0.1.1)

- zh-tw Traditional-only enforcement:
  - strict: fail with traditional_check_failed:<offending_chars>
  - standard: warn
  - dev_repair_ok: run repair task repair_traditional:v1, then re-check

- ja style enforcement (タメ口 default):
  - metrics include a “formal ratio” heuristic (です/ます density)
  - standard: warn
  - dev_repair_ok: run repair task repair_ja_style:v1, then re-check

## First Person Mode (Opinion Mode) + POV Spine (Phoenix Raising)

This system has a production “first person mode” that is implemented as opinion_mode=True plus an optional pov_spine payload. The goal is to produce a controlled first-person author voice (memoir-like, experience-framed), without drifting into practitioner/case-study voice.

### What it is

- **opinion_mode**: A mode flag that changes narrative voice expectations.
- **POV spine (pov_spine)**: A structured set of first-person voice anchors used to keep first-person passages consistent and credible.

The POV spine structure (canonical fields):

- belief
- former_belief
- tension
- refusal
- cost_paid
- boundary

### Where it is configured / passed through

- **Phoenix Raising generator**: SOURCE_OF_TRUTH/tools/phoenix_raising_creator.py
  - PhoenixRaisingCreator.__init__(..., opinion_mode: bool = False, pov_spine: Optional[Dict[str, str]] = None)
- **POV spine generation/validation utilities**:
  - phoenix_ui/utils/pov_spine_generator.py
    - POVSpineGenerator.generate_from_author(...)
    - validate_pov_spine_with_author(...)
  - phoenix_ui/utils/opinion_validator.py
    - OpinionModeValidator.validate_complete_setup(...)

### Deterministic distribution: first-person vs composite story parts

In Phoenix Raising, opinion/first-person mode affects **story voice**, not story placement.

The key rule is a deterministic mix of first-person and composite storytelling:

- **Chapter 1**: all story layers are forced first-person (L1/L2/L3) when opinion_mode=True.
- **Chapters 2..N**:
  - L1 and L3 are first-person in ~30% of chapters (deterministic)
  - L2 is always composite (unless Chapter 1 override)

Implementation:

- PhoenixRaisingCreator._opinion_first_person_flags(chapter_num, num_chapters)
  - For the canonical 12-chapter arc:
    - L1 first-person in chapters {2, 6, 10}
    - L3 first-person in chapters {2, 4, 8, 12}
  - For other chapter counts:
    - Approximates 30% via stable hashing.

### How first-person content is produced (fallback)

If opinion_mode=True and the current chapter/layer is flagged first-person, Phoenix Raising can generate a deterministic first-person story chunk directly from the POV spine:

- PhoenixRaisingCreator._build_opinion_first_person_story_text(...)
  - Uses POV spine fields (former_belief, belief, cost_paid, boundary) plus light scaffolding.
  - Produces a memoir-like first-person block.
  - Avoids “practitioner case study” language.
  - Marks the generated story with metadata:
    - metadata.category = "first_person"
    - metadata.truth_level = "author"

### Opinion-mode enforcement points (voice safety)

Phoenix Raising applies additional voice constraints under opinion_mode=True:

- **Hard filter: block practitioner case-study voice**
  - PhoenixRaisingCreator._contains_practitioner_case_study_language(text)
  - Used during story selection retries to avoid “I worked with a client/patient…” style voice.

- **Replace practitioner-framed author paragraphs in prose body**
  - In _assemble_chapter_with_beats (prose hydration stage), when opinion_mode and pov_spine are present, the system replaces common practitioner-framed paragraphs with a POV-spine “micro-story” assembled from:
    - former_belief, cost_paid, belief, boundary

### Relationship to Narrative Flow opinion mode (atoms)

The broader system also has opinion mode support in the Narrative Flow orchestrator (atom pipeline):

- OPINION_MODE_NARRATIVE_FLOW_IMPLEMENTATION.md
- cluster_therapy_tok/runtime/orchestrator.py
  - Opinion atoms: opinion_{atom_type}
  - Fallback transform: _transform_to_opinion()
  - Validation: _validate_opinion_atom()

Phoenix Raising (CSV pipeline) and Narrative Flow (atom pipeline) both use the same conceptual “opinion/first-person” contract, but enforce it in different places.

### How to verify first person mode is working

- **Input verification**
  - opinion_mode=True is set for the run.
  - pov_spine is present and complete (6 required fields), or you accept reduced strength.

- **Manifest / chapter metadata checks**
  - Look for story objects where metadata.category == "first_person" and IDs like opinion_fp_<chapter>_<layer>.

- **Text output checks**
  - First-person layers contain clear I ... voice and POV-spine phrasing.
  - Practitioner/case-study phrases are absent (e.g., “my client”, “in my practice”, “I worked with someone”).

def assign_mechanism_emphasis(persona_id: str, topic_id: str, book_id: str) -> str:
    """

        'burnout': ['automatic', 'protective'],
        'perfectionism': ['automatic', 'inherited'],
        'anxiety': ['automatic', 'invisible'],
        'people_pleasing': ['protective', 'inherited'],
        'imposter_syndrome': ['invisible', 'inherited'],
        # ... etc

    if topic_id in TOPIC_EMPHASIS_MAP:
        options = TOPIC_EMPHASIS_MAP[topic_id]
        seed = f"{book_id}_emphasis"

    all_emphasis = ['automatic', 'protective', 'invisible', 'inherited']
    seed = f"{book_id}_emphasis"
    hash_val = int(hashlib.sha256(seed.encode()).hexdigest(), 16)
    return all_emphasis[hash_val % len(all_emphasis)]

def assign_motif(structure_id: str, emphasis: str, book_id: str) -> str:
    """
    Assign recurring motif based on structure + emphasis.
    
    Rules:
    - opening structures with 'question' → motif_seeing or motif_recognizing
    - emphasis 'automatic' → motif_automatic or motif_sentence
    - emphasis 'invisible' → motif_invisible or motif_seeing
    - emphasis 'protective' → motif_permission
    """
    
    # Structure + emphasis → motif pairing
    if emphasis == 'automatic':
        options = ['automatic', 'sentence', 'recognizing']
    elif emphasis == 'invisible':
        options = ['invisible', 'seeing', 'pattern']
    elif emphasis == 'protective':
        options = ['permission', 'seeing', 'noticing']
    elif emphasis == 'inherited':
        options = ['sentence', 'pattern', 'seeing']
    else:
        options = ['seeing', 'noticing', 'recognizing', 'pattern']
    
    # Deterministic selection
    seed = f"{book_id}_motif"
    hash_val = int(hashlib.sha256(seed.encode()).hexdigest(), 16)
    return options[hash_val % len(options)]

### **Injector Implementation:**

python
class MotifInjector:
    """Injects recurring motif phrases into designated beats."""
    
    def __init__(self, motif_bank_path: str):
        self.motifs = self._load_motifs(motif_bank_path)
    
    def inject_motif(
        self,
        chapter_content: str,
        chapter_num: int,
        motif_id: str
    ) -> str:
        """
        Inject motif phrases at designated points.
        
        Args:
            chapter_content: Current chapter text
            chapter_num: Chapter number (1-12)
            motif_id: Motif identifier
            
        Returns:
            Modified chapter with motif injected
        """
        motif = self.motifs[motif_id]
        
        # Get injection points for this chapter
        if f'chapter_{chapter_num}' not in motif['injection_points']:
            return chapter_content  # No injection for this chapter
        
        injection_points = motif['injection_points'][f'chapter_{chapter_num}']
        
        # Inject at each point
        for point in injection_points:
            # Find marker in content
            marker = self._get_marker_for_point(point)
            if marker not in chapter_content:
                continue
            
            # Select phrase (rotate through variations)
            phrase = self._select_phrase(motif, point)
            
            # Inject
            chapter_content = self._inject_at_marker(
                content=chapter_content,
                marker=marker,
                phrase=phrase
            )
        
        return chapter_content

class MotifInjector:
    CONSTRAINTS = {
        'max_per_chapter': 2,
        'min_distance_words': 250,
        'max_total_per_book': 12
    }
    
    def inject_motif(self, chapter_content: str, chapter_num: int, motif_id: str) -> str:
        motif = self.motifs[motif_id]
        injection_points = motif['injection_points'].get(f'chapter_{chapter_num}', [])
        
        # Enforce max per chapter
        if len(injection_points) > self.CONSTRAINTS['max_per_chapter']:
            injection_points = injection_points[:self.CONSTRAINTS['max_per_chapter']]
        
        injected_positions = []
        
        for point in injection_points:
            marker = self._get_marker_for_point(point)
            if marker not in chapter_content:
                continue
            
            marker_position = chapter_content.find(marker)
            
            # Enforce minimum distance
            if self._check_distance_constraint(marker_position, injected_positions):
                phrase = self._select_variation(motif, len(injected_positions))
                chapter_content = self._inject_at_marker(chapter_content, marker, phrase)
                injected_positions.append(marker_position)
        
        return chapter_content
    
    def _check_distance_constraint(self, new_pos: int, existing_positions: list) -> bool:
        min_words = self.CONSTRAINTS['min_distance_words']
        for existing_pos in existing_positions:
            word_distance = abs(new_pos - existing_pos) / 5
            if word_distance < min_words:
                return False
        return True

def middle_slice_similarity_test(book_sample: list) -> dict:

                })

    results['max_similarity'] = np.max(results['pairwise_similarities'])

### Biased Selection (NOT Hard-Coding)

python
def apply_series_bias(series_id: str, book_id: str) -> dict:
    """
    Apply series preference bias while maintaining variety.
    
    70% chance of preferred structure, 30% any structure.
    This preserves anti-fingerprinting diversity.
    """
    
    series = load_series_definition(series_id)
    bias_weight = series['selection_bias']['bias_weight']
    
    opening_structure = weighted_select(
        all_options=ALL_OPENING_STRUCTURES,
        preferred=series['selection_bias']['preferred_opening_structures'],
        bias_weight=bias_weight,
        seed=f"{book_id}_opening"
    )
    
    motif = weighted_select(
        all_options=ALL_MOTIFS,
        preferred=series['selection_bias']['preferred_motifs'],
        bias_weight=bias_weight,
        seed=f"{book_id}_motif"
    )
    
    emphasis = weighted_select(
        all_options=ALL_EMPHASES,
        preferred=series['selection_bias']['preferred_emphasis'],
        bias_weight=bias_weight,
        seed=f"{book_id}_emphasis"
    )
    
    origin = weighted_select(
        all_options=ALL_ORIGINS,
        preferred=series['selection_bias']['preferred_origin'],
        bias_weight=bias_weight,
        seed=f"{book_id}_origin"
    )
    
    return {
        'series_id': series_id,
        'opening_structure': opening_structure,
        'motif_id': motif,
        'mechanism_emphasis': emphasis,
        'origin_structure': origin,
        'metadata_tone': series['metadata_tone']
    }

def weighted_select(all_options: list, preferred: list, bias_weight: float, seed: str) -> str:
    """Deterministic weighted selection."""
    import random
    random.seed(hash(seed))
    
    if random.random() < bias_weight and preferred:
        return random.choice(preferred)
    else:
        return random.choice(all_options)

**Why 70/30 split:**
- 70% creates series "feel" and consistency
- 30% maintains structural diversity
- Preserves anti-fingerprinting guarantees
- Still deterministic (same seed = same result)

### Series Overlap Check

def check_series_overlap(series_definitions: dict) -> dict:
    """
    Verify series are distinct enough.
    
    Target: < 30% structure overlap between any two series.
    """
    
    results = {'warnings': []}
    series_list = list(series_definitions.keys())
    
    for i in range(len(series_list)):
        for j in range(i+1, len(series_list)):
            series_a = series_definitions[series_list[i]]
            series_b = series_definitions[series_list[j]]
            
            # Check structure overlap
            structures_a = set(series_a['selection_bias']['preferred_opening_structures'])
            structures_b = set(series_b['selection_bias']['preferred_opening_structures'])
            overlap = structures_a & structures_b
            overlap_pct = len(overlap) / max(len(structures_a), len(structures_b))
            
            if overlap_pct > 0.3:
                results['warnings'].append(
                    f"High overlap ({overlap_pct:.0%}) between {series_list[i]} and {series_list[j]}"
                )
    
    return results
