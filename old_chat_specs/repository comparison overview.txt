# How to Make Sure All Stories Exist in V4

V4 assembly requires **story atoms** for each (persona, topic, role, engine, length) when it injects a micro-story into a slot. If any required combination is missing, assembly fails with "No story_atoms found for this combination."

---

## 1. Where stories must live

**Root:** SOURCE_OF_TRUTH/data/story_bank/story_atoms/

**Layout:**
text
story_atoms/
  {persona}/           e.g. gen_z_la, gen_z_professionals, healthcare_burnout
    {topic}/           e.g. anxiety, burnout, grief, stress
      {engine}/        from topic_engine_bindings.yaml (e.g. avoidance_trap, legacy, false_alarm)
        {role}/        recognition | mechanism_proof | turning_point | embodiment
          micro/       length (V4 uses micro)
            *.txt      one file per variant (e.g. v01.txt, v02.txt)
            *_meta.yaml  optional; status: confirmed | provisional
            metadata.yaml  optional folder-level defaults

**Required roles (all four):** recognition, mechanism_proof, turning_point, embodiment

**Engines per topic:** Defined in phoenix_v4/planning/topic_engine_bindings.yaml. Each topic lists allowed_engines. At least one engine for that topic must have content under story_atoms/{persona}/{topic}/{engine}/{role}/micro/.

**Minimum variants:** For reliable assembly and plan-time coverage, aim for **≥ 5 variants per role** (e.g. 5+ .txt files in that role’s micro/ folder). Format policy can require more (e.g. standard_book in phoenix_v4/policy/format_story_quality.yaml: 10 recognition, 10 mechanism_proof, 6 turning_point, 6 embodiment for confirmed-only).

---

## 2. Check what’s missing

### A. Topic readiness (one persona × topic)

bash
python3 phoenix_v4/ci/check_topic_readiness.py <topic> <persona> --verbose
# Example:
python3 phoenix_v4/ci/check_topic_readiness.py anxiety gen_z_la --min-level RELEASE_READY --verbose

Use --min-level RELEASE_READY to enforce enough atoms for release. Output shows whether story_atoms coverage is sufficient.

### B. Full coverage report (all personas × topics)

bash
python3 phoenix_v4/ci/story_atoms_coverage_report.py --out artifacts/story_atoms_coverage.json

Then inspect artifacts/story_atoms_coverage.json: each persona::topic has status (READY | PARTIAL | MISSING) and missing path hints. **READY** = all required roles have ≥ 5 variants for at least one allowed engine.

### C. Plan-time coverage (when compiling a plan)

compile_format_plan uses phoenix_v4/planning/coverage_checker.py and phoenix_v4/policy/format_story_quality.yaml. If you don’t use --allow-coverage-gaps, it will raise when counts are below the format’s min_atoms_per_role (and when V4_STRICT is on). So running a plan compile for a given (format, persona, topic) and seeing no coverage error is another “stories exist” check.

---

## 3. Fix missing stories

### Option A: Add story atoms

1. Create the directory for the missing (persona, topic, engine, role):
   
text
   SOURCE_OF_TRUTH/data/story_bank/story_atoms/{persona}/{topic}/{engine}/{role}/micro/
2. Add .txt files (one per variant, e.g. v01.txt, v02.txt, …).
3. Optionally add v01_meta.yaml, v02_meta.yaml, … with status: confirmed or status: provisional.
4. Ensure the **engine** is in topic_engine_bindings.yaml under that topic’s allowed_engines.

Use your normal authoring or generation pipeline (e.g. prompt packs from phoenix_v4/agent/system_atom_packs.py, or manual authoring) to create the content.

### Option B: Persona fallback (temporary)

In phoenix_v4/libraries/micro_story_loader.py, PERSONA_FALLBACK maps personas to fallback personas when no atoms exist for the requested persona. Example:

python
PERSONA_FALLBACK = {
    "gen_z_la": ["gen_z_professionals", "gen_z"],
    "gen_z": ["gen_z_professionals"],
}

If the story injector is built with use_persona_fallback=True, it can use another persona’s atoms when the requested (persona, topic, role, …) has no files. That avoids assembly failure but is not a substitute for real coverage. Check where MicroStoryLoader is instantiated (e.g. section_assembler, story_injector) to enable or disable fallback.

### Option C: Restrict campaigns to ready pairs

Run the prose QA (or any batch) only for (persona, topic) pairs that are **READY** in story_atoms_coverage_report.py output, or that pass check_topic_readiness.py ... --min-level RELEASE_READY. Then you don’t hit “No story_atoms found” for those books.

def load_v4_registry_personas(repo_root: Path) -> list[str]:
    """Personas from phoenix_v4/registries/personas (same as prose QA campaign)."""
    reg = repo_root / "phoenix_v4" / "registries" / "personas"
    if not reg.exists():
        return ["gen_z_la", "gen_z_professionals", "healthcare_burnout"]
    out = []
    for f in sorted(reg.glob("*.yaml")):
        try:
            import yaml
            with open(f, "r", encoding="utf-8") as fp:
                d = yaml.safe_load(fp) or {}
            pid = d.get("persona_id") or f.stem
            if pid:
                out.append(pid)
        except Exception:
            continue
    return out if out else ["gen_z_la", "gen_z_professionals", "healthcare_burnout"]


def load_v4_registry_topics(repo_root: Path) -> list[str]:
    """Topics from phoenix_v4/registries/topics (same as prose QA campaign)."""
    reg = repo_root / "phoenix_v4" / "registries" / "topics"
    if not reg.exists():
        return ["anxiety", "burnout", "grief", "stress"]

    for f in sorted(reg.glob("*.yaml")):
        try:
            import yaml
            with open(f, "r", encoding="utf-8") as fp:
                d = yaml.safe_load(fp) or {}
            tid = d.get("topic_id") or f.stem

#!/usr/bin/env python3
"""
Repair legacy story atoms produced by template_atom_writer:
- Remove truncated endings (e.g. "They h" from mid-word cut at 150 words).
- Deduplicate repeated sentences (order-preserving) so each sentence appears once.

Only modifies .txt files under story_atoms/**/legacy/**/micro/*.txt.
"""

from pathlib import Path
import re
import sys

REPO_ROOT = Path(__file__).resolve().parent.parent
ATOMS_ROOT = REPO_ROOT / "SOURCE_OF_TRUTH" / "data" / "story_bank" / "story_atoms"


def _sentences(text: str) -> list[str]:
    """Split on period + space, keep sentence boundaries."""
    text = text.strip()
    if not text:
        return []
    parts = re.split(r"\.\s+", text)
    out = []
    for i, p in enumerate(parts):
        p = p.strip()
        if not p:
            continue
        if i < len(parts) - 1:
            p = p + "."
        out.append(p)
    return out


def _dedupe_sentences(sentences: list[str]) -> list[str]:
    """Order-preserving dedupe."""
    seen = set()
    out = []
    for s in sentences:
        n = s.strip().lower()
        if n in seen:
            continue
        seen.add(n)
        out.append(s)
    return out


def repair_body(body: str) -> str:
    """Fix truncation and repetition. Returns repaired text (60-150 words)."""
    body = body.strip()
    # Remove trailing incomplete fragment (e.g. "They h" or "The " or single letter)
    body = re.sub(r"\s+[A-Za-z]{1,2}\s*$", "", body)
    body = body.strip()
    if body and not body.endswith("."):
        body = body + "."
    sentences = _sentences(body)
    if not sentences:
        return body
    deduped = _dedupe_sentences(sentences)
    joined = " ".join(deduped)
    word_count = len(joined.split())
    if word_count < 60:
        # Keep as-is; we can't add content without template
        return joined
    if word_count > 150:
        # Take first 150 words on word boundary
        words = joined.split()[:150]
        return " ".join(words)
    return joined


def main() -> int:
    repaired = 0
    for txt_path in sorted(ATOMS_ROOT.rglob("*.txt")):
        if "legacy" not in txt_path.parts or "micro" not in txt_path.parts:
            continue
        text = txt_path.read_text(encoding="utf-8")
        if "They h" in text or not text.strip().endswith("."):
            # Possible truncation
            pass
        else:
            # Check for repeated sentences
            sentences = _sentences(text)
            deduped = _dedupe_sentences(sentences)
            if len(deduped) < len(sentences):
                pass  # has repetition
            else:
                continue  # no repair needed
        fixed = repair_body(text)
        if fixed != text:
            txt_path.write_text(fixed, encoding="utf-8")
            repaired += 1
            print(txt_path.relative_to(REPO_ROOT))
    print(f"Repaired {repaired} files.", file=sys.stderr)
    return 0


if __name__ == "__main__":
    sys.exit(main())

**Initial QA findings (before repair):**

| Issue | Cause | Impact |
|-------|--------|--------|
| **Truncation** | Template writer sliced at 150 *words* then joined with `" ".join(word_list)`, which could cut mid-word (e.g. "They h"). | Broken final sentence in many legacy .txt files. |
| **Repetition** | Body was built by repeating the same 10 filler sentences in a loop, so the same sentence appeared 2+ times in one atom. | Felt generic and redundant vs. curated atoms. |

**Curated (non-legacy) atoms** (e.g. same persona/topic under a non-legacy engine, or hand-written) typically have:

- One occurrence per sentence, 60–150 words, third person.

   - Word count capped at 150 on a **word boundary** (no mid-word truncation).

# ---------------------------------------------------------------------------
# Qwen LLM writer (V4 story atoms via local Qwen / OpenAI-compatible API)
# ---------------------------------------------------------------------------

SYSTEM_PROMPT_V4_ATOM = """You are a story-atom writer for a therapeutic book. You write micro-stories (60-150 words) that fit a given persona, topic, and role. Output ONLY the requested format—no preamble or explanation.

Output format (use exactly these labels):
BODY:
<60-150 words, third person only. Experience only; no advice, no "teacher says", no biography. End with a period.>

META:
point: "<one declarative sentence: what this story proves>"
reversal_line: "<one sentence meaning-change hinge>" 
stakes: "<what the character wants in that moment>"

Rules for META:
- point: required. Declarative only (e.g. "The body can signal before the mind names the pattern."). No "you should" or advice.
- reversal_line: required ONLY when role is turning_point or embodiment; otherwise omit the line or leave empty.
- stakes: optional but recommended.

Role semantics:
- recognition: body ends in pattern/tension, no shift.
- mechanism_proof: body shows pattern (e.g. trying something and result).
- turning_point: crack only, not cure; include reversal_line.
- embodiment: action without triumph; include reversal_line."""


def _parse_qwen_atom_response(response: str, role: str) -> Tuple[Optional[str], Optional[Dict[str, Any]]]:
    """Parse LLM response into body and meta dict. Returns (None, None) on parse failure."""
    response = (response or "").strip()
    if not response:
        return None, None
    body_text: Optional[str] = None
    meta_dict: Dict[str, Any] = {}

    # Split on META: (case-insensitive)
    meta_marker = "META:"
    idx = response.upper().find(meta_marker)
    if idx >= 0:
        body_block = response[:idx].strip()
        meta_block = response[idx + len(meta_marker):].strip()
        # Body: strip "BODY:" if present
        if body_block.upper().startswith("BODY:"):
            body_block = body_block[5:].strip()
        body_text = body_block
        # Parse meta: YAML-like key: value (one per line)
        for line in meta_block.splitlines():
            line = line.strip()
            if not line or line.startswith("#"):
                continue
            if ":" in line:
                k, v = line.split(":", 1)
                k = k.strip().lower().replace("-", "_")
                v = v.strip().strip('"\'')
                if k == "reversal_line" and role not in ("turning_point", "embodiment"):
                    continue
                meta_dict[k] = v
    else:
        # No META: try whole response as body, no meta
        if response.upper().startswith("BODY:"):
            body_text = response[5:].strip()
        else:
            body_text = response

    point = (meta_dict.get("point") or "").strip()
    if not body_text or not point:
        return None, None
    # Enforce 60-150 words on body
    words = body_text.split()
    if len(words) > 150:
        body_text = " ".join(words[:150])
    if len(words) < 60 and body_text:
        pass  # accept short body; lint may still reject
    meta_dict.setdefault("point", point)
    meta_dict.setdefault("reversal_line", None)
    meta_dict.setdefault("stakes", "")
    return body_text, meta_dict


def make_qwen_atom_writer(
    base_url: str = "http://localhost:1234/v1",
    model: str = "Qwen2.5-14B-Instruct",
    api_key: str = "lm-studio",
    timeout: float = 90.0,
) -> Callable[[str, Dict[str, str]], Tuple[Optional[str], Optional[Dict[str, Any]]]]:
    """Build a writer that calls local Qwen (OpenAI-compatible API) to generate body + meta per pack."""

    def qwen_atom_writer(pack_text: str, parsed: Dict[str, str]) -> Tuple[Optional[str], Optional[Dict[str, Any]]]:
        role = (parsed.get("role") or "").strip().lower()
        try:
            from openai import OpenAI
        except ImportError:
            raise RuntimeError("openai package required for Qwen writer. pip install openai")
        client = OpenAI(base_url=base_url, api_key=api_key, timeout=timeout)
        messages = [
            {"role": "system", "content": SYSTEM_PROMPT_V4_ATOM},
            {"role": "user", "content": pack_text},
        ]
        resp = client.chat.completions.create(
            model=model,
            messages=messages,
            temperature=0.7,
            max_tokens=600,
        )
        choice = resp.choices[0] if resp.choices else None
        if not choice or not getattr(choice, "message", None):
            return None, None
        raw = (choice.message.content or "").strip()
        body, meta = _parse_qwen_atom_response(raw, role)
        if not meta:
            return None, None
        if role in ("turning_point", "embodiment") and not (meta.get("reversal_line") or "").strip():
            meta["reversal_line"] = "They realized the feeling was not a verdict."
        return body, meta

    return qwen_atom_writer

from pathlib import Path
import yaml

FAMILIES_ROOT = Path("phoenix_v4/templates/families")

def load_template_family_prose(family_id: str) -> str | None:
    family_path = FAMILIES_ROOT / family_id
    if not family_path.exists():
        return None

    # A) Directory of ordered .txt files
    if family_path.is_dir():
        txts = sorted(family_path.glob("*.txt"))
        if txts:
            return "\n\n".join(p.read_text().strip() for p in txts)

        # B) YAML with embedded prose
        yamls = list(family_path.glob("*.yaml"))
        if yamls:
            data = yaml.safe_load(yamls[0].read_text())
            if isinstance(data, dict) and "prose" in data:
                return data["prose"].strip()

    # C) Single file (family.txt / family.yaml)
    if family_path.with_suffix(".txt").exists():
        return family_path.with_suffix(".txt").read_text().strip()

    if family_path.with_suffix(".yaml").exists():
        data = yaml.safe_load(family_path.with_suffix(".yaml").read_text())
        if "prose" in data:
            return data["prose"].strip()

    return None

2️⃣ Auto-Lock Extractor
📄 tools/template_eval/extract_sentence_locks.py


import re

LOCK_PATTERNS = [
    r"nothing is wrong with you",
    r"not something to fix",
    r"you are not broken",
    r"this is not a problem",
]

def extract_sentence_locks(text: str):
    sentences = re.split(r'(?<=[.!?])\s+', text)
    locks = []

    for i, s in enumerate(sentences):
        for pat in LOCK_PATTERNS:
            if re.search(pat, s.lower()):
                locks.append({
                    "id": f"AUTO-{i:03d}",
                    "text": s.strip(),
                    "severity": "CRITICAL"
                })
                break
    return locks
