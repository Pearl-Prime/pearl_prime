# I. MECHANISM ESCALATION TRACKING

## Problem

Current V4 ensures:

* mechanism_proof role exists per chapter.

But it does NOT enforce:

* increasing depth of understanding across chapters.

Books risk feeling:

* flat in insight progression
* repetitive in mechanism explanation

---

## Upgrade: Mechanism Depth Levels

Add to atom metadata:

```yaml
metadata:
  mechanism_depth: 1   # 1=surface, 2=behavioral, 3=nervous_system, 4=identity
```

### Required Escalation Rule (Per Arc)

For a 10-chapter book:

| Phase | Required Depth |
| ----- | -------------- |
| Early | depth 1–2      |
| Mid   | depth 2–3      |
| Late  | depth 3–4      |

Planner must enforce:

```
max(mechanism_depth in chapter N) 
    >= max(mechanism_depth in chapter N-1)
```

No regression allowed.

---

## CI Gate

Add `mechanism_escalation_gate.py`

Fail if:

* Depth plateaus after midpoint
* Depth decreases late-stage
* No depth-4 identity-level mechanism by final third

# II. COST GRADIENT ENFORCEMENT

## Problem

V4 enforces presence of stakes.
It does NOT enforce escalation of cost.

Narratives need:

* social cost
* internal cost
* avoidance cost
* identity cost

---

## Upgrade: Cost Markers

Add to scenes + atoms:

```yaml
metadata:
  cost_type: social | internal | opportunity | identity
  cost_intensity: 1-5
```

---

## Required Arc Gradient

For mid-length book:

```
Chapter 1–2: cost_intensity 1–2
Chapter 3–6: cost_intensity 2–4
Chapter 7–9: cost_intensity 4–5
Final chapter: resolution without erasing cost history
```

Planner enforces:

```
avg(cost_intensity per chapter) must increase through midpoint
```

---

## CI Gate

`cost_gradient_gate.py`

Fail if:

* Highest cost occurs before midpoint
* No high-intensity cost before identity shift
* Cost disappears too early

# III. CALLBACK SLOT ENFORCEMENT

## Problem

Scene Injection allows callbacks conceptually.
It does not enforce symbolic continuity.

---

## Upgrade: Callback IDs

Scenes gain:

```yaml
metadata:
  callback_id: "a_train_mirror"
  callback_phase: setup | escalation | return
```

Planner must enforce:

* If a callback_id appears in early chapter,
  it must reappear in later chapter with callback_phase=return.

Add required callback slot in arc plan:

```yaml
arc_requirements:
  requires_callback: true
  callback_min_count: 2
```

---

## CI Gate

`callback_integrity_gate.py`

Fail if:

* Setup exists without return
* Return exists without setup
* More than 2 unclosed symbolic threads
