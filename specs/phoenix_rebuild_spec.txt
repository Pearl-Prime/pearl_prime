# DEV SPEC — V4 Exercise System (Canonical, Slot-Compatible, Template-Safe)

## Goal

Rebuild V4 exercise handling as a **first-class structured content subsystem** that:

1. Preserves old V4 exercise structure:

   * **intro → guided_practice → aha_noticing → integration**
2. Supports persona/topic/location style without hardcoding proper nouns.
3. Slots exercises into V4 chapter templates deterministically.
4. Enforces quality via **lint + CI gates** (no resolution, no promises, no “bullet-point therapy”).
5. Allows human-authored exercises + optional model-assisted repair, with explicit approval.

**Non-goal:** auto-generating entire exercises from atoms without review.

# 3) Selection & Slotting Rules

## 3.1 V4 template integration

In V4, exercises are “blocks” that fill **exercise slots** (existing slot_07_practice or expanded).

### Required interface

Add an ExerciseResolver that returns an exercise block string (or structured sections) to assembly.

#### Resolver inputs

* `persona_id`
* `topic_id`
* `chapter_num`
* `template_family`
* `slot_id` (e.g. `slot_07_practice`)
* `location_context` (optional, for overlay hydration)
* `seed` (book_id deterministic seed)
* `constraints` (intensity, duration max, modality include/exclude)

#### Resolver output

* selected `exercise_id`
* rendered `content` (with placeholders intact OR hydrated depending on stage)

---

## 3.2 Deterministic choice algorithm (must)

**No randomness without seed.** Use book_id + chapter + slot for stable selection.

Pseudo:

1. Filter approved exercises by persona/topic.
2. Apply template-family filters (e.g. grief disallows auto-promote, but selection still allowed).
3. Apply constraints (intensity, modalities, duration).
4. Enforce **no adjacent reuse** across chapters.
5. Choose deterministically:

   * `hash(book_id + chapter_num + slot_id) % len(pool)`

Add rotation behavior:

* Prefer exercises not used in previous N chapters.

---

## 3.3 Slot policies

Add `docs/EXERCISE_SLOT_POLICY.yaml`:

```yaml
slot_07_practice:
  required: true
  allowed_modalities: [breath, body_scan, grounding, visualization, journaling]
  allowed_intensity: [gentle, medium]
  max_duration_seconds: 240
  cadence_role: [grounding, release, integration]
```

Templates can override:

```yaml
template_overrides:
  bestseller_v2_1:
    slot_07_practice:
      allowed_intensity: [gentle]
      cadence_role: [grounding]
```

# 4) Lint + Quality Gates (Hard)

## 4.1 Lint checks (exercise-level)

Create: `tools/exercise_lint/lint_exercise.py`

Outputs:

```py
class LintResult:
  passed: bool
  errors: list[str]
  warnings: list[str]
```

### Required checks

1. **Schema validity**

   * required keys present
   * correct persona/topic ids exist
2. **Structure**

   * all 4 sections non-empty
   * min word counts per section (configurable)
3. **Voice**

   * second-person dominant (“you”)
   * present tense heuristic
4. **No resolution / no cure**

   * classifier/regex for “you will feel better / this will heal / you are cured”
   * disallow “fixed, cured, healed” in closing claims
5. **No unsafe therapy claims**

   * no medical guarantees
6. **Cadence**

   * intro cannot jump immediately to steps without emotional safety line
   * integration cannot “wrap it up” with a conclusion statement
7. **Placeholder integrity**

   * if placeholders exist, must be from allowed location overlay schema
8. **Style blacklist**

   * forbid toxic positivity, shame language, “just…”

### Reject logging

Write:

* `SOURCE_OF_TRUTH/exercises_v4/_rejects/<exercise_id>.log`
* include errors + offending text snippet

# 6) Approval CLI (Tiny Tool)

Create: `tools/exercise_approval/exercise_approve.py`

### Commands

```bash
# approve one
python3 tools/exercise_approval/exercise_approve.py approve \
  --id ex_90_second_reset \
  --by "olya" \
  --note "Fits gen_alpha anxiety; strong intro"

# reject one (moves to _rejects, sets status)
python3 tools/exercise_approval/exercise_approve.py reject \
  --id ex_x \
  --by "olya" \
  --note "Too outcome-y in integration"

# auto-promote (batch) with safeguards
python3 tools/exercise_approval/exercise_approve.py auto-promote \
  --persona gen_alpha \
  --topic anxiety \
  --min-score 0.9 \
  --max-count 50
```

### Where does score come from?

If you already have template_eval scoring, add `exercise_eval` OR keep auto-promote purely lint + heuristic:

* “auto-promote only if lint passes + zero warnings + cadence ok”
* optionally require a “golden exemplar similarity” score later
