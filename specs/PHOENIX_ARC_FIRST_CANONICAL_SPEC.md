# 1. CORE ARCHITECTURE

Three authority layers:

1. Arc Blueprint (mandatory)
2. Engine Overlay
3. Atom Pools

There is no quality simulation layer.

---

# 2. ARC BLUEPRINT (MANDATORY)

## 2.1 Arc Requirement

Every compile requires:

* arc_id
* persona
* topic
* engine
* format

No arc = no compile.

All compiles must explicitly reference an arc file.

---

## 2.2 Arc Authorship & Bootstrap

Arcs are:

* Authored manually
* Human-designed
* Persona × Topic × Engine × Format specific

The system does NOT generate arcs.

Bootstrapping rule:

If compiling new combinations:

* Author arc first
* Commit arc
* Then compile

Estimated scale:
300–500 arcs to support 10k+ books via atom variation.

Arcs are reusable across variants.
They are not autogenerated.

---

## 2.3 Arc Schema

Arc defines:

* emotional_curve (per chapter BAND)
* emotional_temperature_curve
* chapter_intent
* reflection_strategy_sequence
* cost_chapter_index
* motif
* resolution_type

No prose inside arc file.

---

# 3. ARC VALIDATION — STRUCTURAL ONLY

## 3.1 What validate_arc_alignment sees

Inputs:

* Compiled Plan (slot assignments)
* Arc Blueprint file used for compile

It does NOT inspect prose.

It verifies:

* Chapter BAND matches emotional_curve
* Reflection strategy matches sequence
* Exactly one cost_confrontation chapter exists
* Cost chapter index matches arc
* Ending integration respects resolution_type
* Engine permits resolution_type

It does NOT:

* Score intensity
* Evaluate prose quality
* Detect tone
* Simulate emotional variance

---

# 4. ENGINE OVERLAY

Each topic has one engine.

Engine defines:

* Allowed resolution_types
* Open-loop permissions
* Identity shift permissions
* Tone constraints

If Arc resolution_type violates Engine → compile fails.

Engine governs behavior.
Arc governs pacing.

---

# 5. COST CHAPTER CLARIFICATION

Arc defines:

* cost_chapter_index

Format defines:

* Cost chapter slot sequence
* Beat structure

Cost chapter uses fixed format beat map.

Arc chooses WHICH chapter.
Format defines HOW it is structured.

Arc does not redefine cost slot sequence.

---

# 6. REFLECTION STRATEGY GOVERNANCE

Arc declares:

* reflection_strategy_sequence

Compiler enforces exact match.

No auto-rotation.
No auto-balancing.
No dynamic strategy logic.

---

# 7. LOCATION IMMERSION

Arc may declare:

* locale_key
* immersion_density

Compiler ensures SCENE slot presence.
Compiler does not score scene richness.

Location quality is writer-owned.
immersion_density is advisory (e.g. for renderer or docs only); no validator enforces it.

---

# 8. RESOLUTION TYPE (RECONCILED)

Arc must define:

* resolution_type

Allowed:

* open_loop
* internal_shift_only
* grounded_reframe
* identity_shift

Engine restricts allowed types.

Compiler checks:

* Ending Integration alignment
* STORY atoms do not contradict resolution boundary

This resolves prior shame engine conflict.
