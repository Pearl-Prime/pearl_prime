Vision

Brand admins operate a publishing system that spans lanes (self-help, manga, Pearl News, tools), markets, and proof-backed packaging. This onboarding path shows real outputs — honestly labeled when assets are still pending.

How it works

System spine

  • Registry — config/onboarding/example_registry.json lists every proof tile, comparison set membership, and status (ready / planned / missing).
  • Gallery & matrix — Static pages validate the model locally and on Pages before the wizard deepens choices.
  • Wizard — React app (brand-wizard-app) surfaces matching example IDs after persona / format decisions using the same JSON.
What brand admins do

Your operating loop

Choose narrative and primary reader in the wizard, confirm format bias (book vs manga-forward), then use the gallery to align stakeholders on proof quality. Use the matrix to see market × lane coverage before greenlighting spend.
