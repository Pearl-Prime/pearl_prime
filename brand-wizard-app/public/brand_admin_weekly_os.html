Packaging QA

Validate captions vs asset types; run stale-ready spot checks (broken URLs fall back to proof-pending).

Thu

Distribution sync

Confirm metadata and variant rules per channel; no silent failures on missing assets.

Fri

Stakeholder narrative

Use gallery boards in reviews — teach tradeoffs with comparison sets.

Sat–Sun

On-call judgment

Escalations only; automation handles routine regenerations where pipelines are wired.

Human judgment

  • Positioning and persona nuance
  • When to ship proof-pending vs block launch
  • Civic / editorial tone for Pearl News

Automation

  • Registry-driven galleries and wizard hints
  • Schema validation in CI
  • Build-time sync of onboarding JSON to the app

KPI stubs (fill when wired)

  • Proof coverage % by lane
  • Time from planned to ready
  • Comparison sets complete vs partial

Common failure modes

  • Drift between WP copy and Cloudflare-hosted JSON — pick one source of truth per URL.
  • Treating supporting visuals as product covers — caption discipline matters.
  • Stale ready rows with dead assets — UI must fall back gracefully (see output proof system doc).
