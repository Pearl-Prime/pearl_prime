position. The V1 heuristics (narration → whisper bubble at top; first

## V1 caveats — read before publishing

This is a V1 baseline ship. Expected gaps closed by V2 per the hybrid
character-individuation spec:

- **Identity drift across panels** — same character may look subtly
  different across panels because there's no character-design lock. V2
  closes via 12-axis vocabulary + constraint solver + PuLID-FaceNet
  reference image + face-distance QA harness. Tracked by
  `ws_manga_layered_pipeline_v2_20260507`.
- **Manga register oscillation** — schnell-correct config (4 steps,
  cfg 1.0, euler/simple) mitigates the 28-step audit defect, but the
  seinen-photorealistic register still depends on prompt language alone.
  V2 brand-style LoRA + `animeoutlineV4` register LoRA + halftone
  post-processing layer close this further.
- **ja-JP translation is Tier 1 baseline** — generated by the same
  operator-present session that authored en-US. Native-speaker review
  recommended pre-WEBTOON-Japan publish.
- **Bubble placement is heuristic** — narration → top whisper bubble,
  first dialogue → bottom-left, etc. No layout-engine awareness of
  character face position, panel composition, or reading-flow tension.
  V2's lettering layer wires `genre_prompt_cookbook_v2.yaml` plus the
  per-genre drawing-tradition lookup.

These are V1's known shape — not bugs. Ship V1, gather operator and
reader response, then V2 ships the upgraded re-renders alongside the
historical V1 archive.
