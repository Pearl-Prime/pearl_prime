## Files involved

Pearl News is a Python content pipeline that publishes WordPress posts. It does **not** have separate templates / theme files in this repo — the layout is generated by a single Python module that emits HTML + inline `<style>` blocks, scoped under `.pn-article-root` to avoid leaking into the WordPress (Newspaper) theme.

## "Regression commit" — there is none on main

There is no commit on `main` that removed the `editorial` / `wide` layouts. **The layouts never landed on `main`.** This contradicts the operator's framing of "regression," and matters for choice of strategy.

Evidence:
- `git log --all --follow -S 'CSS_EDITORIAL' -- pearl_news/pipeline/assemble_v52.py` returns exactly **one commit** (`d6ead3af7a`, the addition). If a removal commit existed, `git log -S` would show two commits (one positive Δ, one negative Δ).
- `git branch --contains d6ead3af7a8798c5c2d7da49f2052d293e2f44d4` → only `agent/fix-pearl-news-deterministic-slots` and its remote.
- `git rev-list origin/main | grep d6ead3af7a` → 0 hits (not in main ancestry).
- `origin/main`'s most recent commit on `pearl_news/pipeline/assemble_v52.py` is `3d8fd2ae9a7cac86fd53e217ff8d25499b20cae6` (2026-04-23 09:56), which **predates** the layout-adder commit.

The likely cause of the "mis-fixed >10 times" pattern: prior agent attempts read the symptom (articles bottom-stacked at desktop), inspected `main`, found only `default | scroll_story | dock`, and concluded that the bottom-stacked rendering must be a CSS bug in `default`'s mobile-collapse `@media (max-width: 768px)` rule. They patched media-query thresholds, container widths, etc. None of those patches address the actual gap — that there is no `wide` (intentional bottom-strip) variant on `main` for the publisher to select.

The mobile-collapse `.sidebar { order: 2 }` rule on `default` is **not** a "bottom variant"; it's a fallback for narrow viewports. WordPress (Newspaper theme) renders the post within a constrained content column, which can make the article container narrower than 768px → the @media query fires → sidebar collapses below content → article body inherits the narrow column → looks like "stuck in bottom variant with collapsed real estate." Symptom matches.
