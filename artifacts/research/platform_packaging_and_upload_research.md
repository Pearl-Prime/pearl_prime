## Western Ebook Platforms

### 1. Amazon KDP (Kindle Direct Publishing)

**Marketplaces:** US, UK, DE, FR, ES, IT, NL, JP, BR, CA, MX, AU, IN + more (14 total storefronts)

#### Upload Requirements

| Field | Requirement |
|-------|------------|
| **Manuscript formats** | EPUB (preferred), DOCX, KPF (Kindle Package Format), HTML. MOBI deprecated as of March 2025. |
| **Cover image** | 2560 x 1600 px (ideal), minimum 1000 x 625 px. Height/width ratio of 1.6:1. JPG or TIFF only. RGB color space. 300 DPI minimum. Max 50 MB. |
| **Internal cover** | Required inside EPUB. Must be high-resolution or QA will reject. |
| **Title + subtitle** | Combined max 200 characters. |
| **Description** | Max 4000 characters including HTML tags. Supports limited HTML: bold, italic, lists, H4-H6 headings. No H1-H3. Roughly 300-400 visible words after markup. |
| **Keywords** | 7 keyword phrases, each up to 50 characters. Not displayed on product page but critical for search indexing. |
| **Categories** | 3 Amazon store categories (updated from 2 BISAC codes). Metadata-driven assignment: BISAC codes carry ~80% influence, backend keywords ~65%. |
| **ISBN** | Not required for ebooks. Required for paperback. KDP assigns free ASIN. |
| **AI disclosure** | Mandatory. Must declare AI-generated text, images, or translations. |
| **File size** | Max 650 MB for ebook. |

**Sources:**
- https://kdp.amazon.com/en_US/help/topic/G200634390
- https://kdp.amazon.com/en_US/help/topic/G200645690
- https://kdp.amazon.com/en_US/help/topic/G201189630
- https://kdp.amazon.com/en_US/help/topic/G201097560

#### Pricing & Royalties

| Royalty Tier | Price Range | Delivery Fee | Notes |
|-------------|-------------|--------------|-------|
| **70%** | $2.99 - $9.99 | $0.15/MB | Must be 20% below physical edition price. In-copyright works only. |
| **35%** | $0.99 - $200.00 | None | Public domain works limited to this tier. |
| **KDP Select (70%)** | $2.99 - $9.99 | $0.15/MB | Required for 70% in BR, IN, JP, MX. 90-day exclusivity lock-in. |

**Source:** https://kdp.amazon.com/en_US/help/topic/G200644210

#### Bulk Upload / API

- **No official API.** KDP does not offer a publisher API.
- **Chrome extensions:** KDP Uploader and KDP Uploader Plus automate form filling from Excel spreadsheets. Free tier available. No credential sharing needed.
- **BookUploadPro:** Multi-platform uploader with KDP support, mapped fields, repeatable workflows.
- **Aggregator path:** Draft2Digital, PublishDrive, and StreetLib can distribute to KDP (though most authors upload direct for better royalties).

**Sources:**
- https://kdpcommunity.com/s/question/0D5f400000FHT6ZCAX/kdp-distribution-developers-api
- https://bookuploadpro.com/
- https://www.kdpuploader.com/

#### Content Guidelines

- No public domain content at 70% royalty
- No misleading metadata
- AI-generated content must be disclosed
- Content must comply with KDP content policies (no illegal content, no offensive material)

**Source:** https://kdp.amazon.com/en_US/help/topic/G200672390

### 2. Apple Books (Apple Books for Authors / iTunes Connect)

#### Upload Requirements

| Field | Requirement |
|-------|------------|
| **Manuscript format** | EPUB 3.3 (backward compatible with earlier EPUB 3). Must pass EPUBCheck. |
| **EPUB structure** | Uncompressed mimetype at root, META-INF/container.xml, content folder (OEBPS) with XHTML, CSS, images, fonts, nav document. |
| **Cover image** | Embedded inside EPUB + uploaded separately for storefront. Must not be blurry/pixelated. Must contain title and author name. Recommended 1600 x 2400 px minimum. |
| **Interior images** | Max 5.6 million pixels per image. JPEG or PNG with correct MIME types. |
| **Metadata** | Title, author, language, identifiers (ISBN or unique ID). Author names on cover must exactly match metadata. |
| **Description** | Detailed description required for storefront display. |
| **ISBN** | Recommended but not strictly required (Apple can assign identifier). |
| **AI disclosure** | Required. Use "AI Generated by" artist role. Declare AI creation method in description. |

**Sources:**
- https://help.apple.com/itc/booksassetguide/en.lproj/static.html
- https://help.apple.com/itc/applebooksstoreformatting/en.lproj/static.html
- https://authors.apple.com/support/4574-publish-book-from-web

#### Pricing & Royalties

| Feature | Detail |
|---------|--------|
| **Royalty rate** | 70% on all sales, regardless of price point |
| **Minimum price** | No minimum |
| **Maximum price** | No maximum (still 70% at $24.99+) |
| **Exclusivity** | None required |
| **Hidden fees** | None |

**Source:** https://authors.apple.com/

#### Bulk Upload / API

- **Apple Books for Authors web portal:** Manual upload through web interface.
- **iTunes Transporter:** Command-line tool for bulk metadata and asset delivery. Supports ONIX.
- **Aggregator path:** Draft2Digital, PublishDrive, StreetLib, IngramSpark all distribute to Apple Books.
- **Digital narration:** Apple offers AI narration for eligible English-language reflowable ebooks through preferred partners.

**Source:** https://authors.apple.com/publish

### 3. Google Play Books (Partner Center)

#### Upload Requirements

| Field | Requirement |
|-------|------------|
| **Manuscript formats** | EPUB (preferred) and PDF. Google recommends providing both. |
| **Cover image** | JPEG, PNG, TIFF, or PDF. Minimum 1600 x 2560 px. Square recommended. Separate file upload even if embedded in EPUB. |
| **File naming** | Name EPUB files using electronic ISBN. |
| **File size** | Max 2 GB per file. |
| **Metadata** | Title, author(s), description, subtitle. ONIX or CSV format for bulk. |
| **ISBN** | Own ISBN or free Google Book ID. |
| **Categories** | At least one genre required. All genres from same subject standard (BISAC/BIC). |

**Sources:**
- https://support.google.com/books/partner/answer/3424254
- https://support.google.com/books/partner/answer/9261355

#### Pricing & Royalties

| Feature | Detail |
|---------|--------|
| **Royalty rate** | 52% of list price (standard) or 70% for agency-model publishers |
| **Pricing** | Publisher sets list price. Google may discount but royalty based on list price. |

#### Bulk Upload / API / Automation

- **ONIX feed via SFTP:** Automated content fetching. Publishers provide ONIX 2.0/2.1/3.0 files on HTTP/HTTPS/SFTP/FTP/Google Cloud Storage server. Google fetches on schedule.
- **CSV metadata:** Alternative to ONIX. Can combine ebook and audiobook records.
- **Google-provided SFTP dropbox:** For direct file upload.
- **Bulk processing:** Supports zipped archives.
- **This is the most automation-friendly platform for ebooks.**

**Sources:**
- https://support.google.com/books/partner/answer/2763162
- https://support.google.com/books/partner/answer/6374180

## Audiobook Platforms

### 21. ACX / Audible (Amazon)

#### Audio Requirements

| Spec | Requirement |
|------|------------|
| **Format** | MP3, 192 kbps or higher, constant bit rate (CBR) |
| **Sample rate** | 44.1 kHz |
| **Channels** | Mono preferred (unless content requires stereo). No mixing within same audiobook. |
| **RMS** | Between -23 dB and -18 dB |
| **Peak amplitude** | Under -3 dB |
| **Noise floor** | -60 dB or lower |
| **Consistency** | Pace, tone, volume, quality must be consistent throughout |

#### File Organization

| Spec | Requirement |
|------|------------|
| **Structure** | One chapter/section per file |
| **Section headers** | Must be included in audio (e.g., "Chapter 1" spoken) |
| **Retail sample** | 1-5 minutes, starts with narration (no credits/music) |
| **Cover art** | Min 1000 x 1000 px, JPG or PNG |
| **Metadata** | Title, description, keywords. Must match book cover metadata exactly. |

#### Royalties

| Model | Royalty |
|-------|--------|
| **Exclusive (Audible only)** | 40% |
| **Non-exclusive** | 25% |
| **Royalty Share** | Split with narrator (typically 50/50 of royalty) |

**QA Process:** Semi-automated with AI-driven audio scanners. Rejections can delay weeks.

**Sources:**
- https://help.acx.com/s/article/what-are-the-acx-audio-submission-requirements
- https://narrationbox.com/blog/acx-audio-specs-explained-2025-2026

### 25. Apple Books Audiobooks

#### Requirements

| Spec | Requirement |
|------|------------|
| **Formats** | MP3 (192kbps+, 44.1kHz, CBR) or M4B |
| **Channels** | Stereo recommended |
| **Quality** | High quality, minimal background noise |

#### Upload Path

- Draft2Digital
- Ingram CoreSource
- PublishDrive

## Key Findings Summary

### Critical Facts for System Design

1. **EPUB 3 is the universal format.** Every major platform accepts it. Build once, distribute everywhere.

2. **Cover images need 3 variants:**
   - 2560 x 1600 px (1.6:1 ratio) for Amazon KDP
   - 1600 x 2400 px (3:4 ratio) for Kobo, Apple, general
   - 2560 x 2560 px (square) for Google Play, audiobook platforms

3. **Amazon KDP has no API.** Chrome extensions (KDP Uploader) are the only automation path.

4. **PublishDrive is the only aggregator with a documented open API.** This is the backbone of our automation strategy.

5. **Google Play supports ONIX feed via SFTP** for full automation of ebook + audiobook delivery.

6. **AI disclosure is now mandatory** on Amazon KDP and Apple Books. Include in all metadata packages.

7. **China platforms are blocked for foreign publishers.** Requires licensed Chinese publishing partner. Not automatable from outside China.

8. **LINE Manga and Piccoma are webtoon platforms,** not traditional ebook platforms. Different content type entirely.

9. **Apple Books audiobooks have no direct upload path.** Must use aggregator (D2D, PublishDrive, INaudio).

10. **IngramSpark is raising prices in Feb 2026.** Factor into cost projections.
