## What Broke

### CRITICAL FAILURES

**1. `--author adi_da` flag crashes pipeline**
- Stage: book pipeline preflight
- Error: `ValueError: author_id 'adi_da' not found in author_registry.yaml`
- Root cause: `config/author_registry.yaml` only contains pen-name authors (luna_hart, kai_nakamura, marcus_cole, diane_reyes). Adi Da is a teacher, not a pen-name author. The task spec used `--author` but the correct flag is `--teacher`.
- Fix needed: Document in `docs/EBOOK_PIPELINE_COMPLETE_GUIDE.md` that `--teacher` is for real teachers (with teacher_banks) and `--author` is for pen-name identities only. The spec author confusion caused this.

**2. CosyVoice2 TTS API failure — all modes fail**
- Stage: audiobook tts_render
- Error: All `/api/v1/tts`, natural language control, and 3s-clone modes return `error: null`
- Root cause: CosyVoice2 at 192.168.1.112:9880 is running as a Gradio app. The `/api/v1/tts` endpoint (used by `scripts/audiobook/generate_showcase_bundle.py`) does not exist. The Gradio `/generate_audio` endpoint returns errors for all modes. The Gradio dropdown shows `['']` — no pre-trained voices loaded in the server's voice registry. The 3s-clone and NL-control modes also fail (likely model loading or VRAM issue on Pearl Star).
- Fallback used: `edge_tts` (en-US-ChristopherNeural) for all 12 chapters.
- Fix needed: (1) Load pre-trained voices into CosyVoice2 server. (2) Update `cosyvoice_builtin_tts()` in `generate_showcase_bundle.py` to use Gradio API format instead of `/api/v1/tts`. (3) Provide reference WAV clips for Adi Da voice clone.

**3. Word count gap — 6-hour format not implemented**
- Target: 54,000 words (6h @ 150 WPM)
- Actual: 4,657 words (~30-minute read)
- Root cause: `deep_book_6h` format is defined in `config/catalog/catalog_generation_config.yaml` but not wired to `scripts/run_pipeline.py` as a runtime-format selector. The pipeline uses the section registry to render 1 variant per section, producing ~400 words/chapter regardless of target duration. No `--format deep_book_6h` flag exists in `run_pipeline.py`.
- Fix needed: Implement format-based word-count expansion in the pipeline (e.g., render multiple variants per section, add interstitial content from teacher bank atoms, or wire a "long-form expansion" pass using Qwen).

**4. Podcast pipeline picked wrong content**
- Stage: assemble
- Error: Pipeline assembled "The Room Full of People — Anxiety" (social_anxiety arc) instead of our self_worth book
- Root cause: `scripts/podcast/assemble_podcast_episode.py` reads from the weekly catalog queue (`config/catalog/weekly_queue_config.yaml`) and pulls the next scheduled episode for the brand, not from the specific book just generated. There is no `--book-id` or `--override-content` flag to force specific content.
- Workaround: Manual episode generated from chapter_01.mp3.
- Fix needed: Add `--book-dir` content override to `assemble_podcast_episode.py` that uses the provided book's narration segments rather than the weekly catalog pull.

**5. Video pipeline VCE stages not runnable standalone**
- Stage: script_prep (video pipeline)
- Error: `prepare_script_segments.py` requires `render_manifest.json` (positional arg) and `-o/--out` — no simple `--book` shortcut exists
- Root cause: Video pipeline is designed for pre-generated render manifests (from catalog workflow), not for ad-hoc book → video conversion. No bridge script converts a `book.txt` to `render_manifest.json`.
- Workaround: Manual FFmpeg pipeline (FLUX bg + audio concat).
- Fix needed: Create `scripts/video/book_to_render_manifest.py` that converts a rendered book.txt + chapter audio files into the render_manifest.json format the VCE expects.

### MINOR FAILURES / GAPS

**6. flux_client.py sends `_meta` node — 400 error**
- Stage: ComfyUI image generation
- Error: `HTTP Error 400: Bad Request` — workflow has `_meta` node with no `class_type`
- Root cause: `comfyui_workflows/flux_video_bank.json` has a top-level `_meta` key that ComfyUI treats as a workflow node. The `flux_client.py` doesn't strip it before sending.
- Fix: `if '_meta' in workflow: del workflow['_meta']` in `call_comfyui()`.

**7. `adi_da` not in `pen_name_teacher_profiles_full.json`**
- Checked: 0 matches for `adi_da` in `config/authoring/pen_name_teacher_profiles_full.json`
- Root cause: Adi Da is a teacher identity (SOURCE_OF_TRUTH/teacher_banks/adi_da/ exists), not a pen-name author. The profiles JSON is for pen-name authors only.
- Impact: No author bio, why_this_book, authority_position, or audiobook_pre_intro available for Adi Da from the author asset system.

**8. Brand mapping discrepancy: heart_balance ≠ adi_da in brand_identity_system.yaml**
- `config/catalog_planning/brand_identity_system.yaml`: `heart_balance` → teacher: maat
- `scripts/audiobook/generate_showcase_bundle.py`: Adi Da row → brand: heart_balance
- Impact: `generate_bestseller_covers.py --brand heart_balance` generates "Scales of Ma'at" covers (maat's book), not Adi Da's book. Cover script used `_generate_comfyui_bg` falls back to brand-level metadata which resolves to maat.
- Fix needed: Either update `brand_identity_system.yaml` to add adi_da's brand mapping, or create a separate brand for Adi Da, or pass explicit teacher metadata to the cover script.

**9. EI v2 scoring not run**
- The `--ei-v2-compare` flag was not in the corrected run (only used in the first failed `--author` run). EI v2 composite score not available for this book.
- Fix: Rerun with `--ei-v2-compare` after the author/teacher flag is resolved.

**10. Video subtitles not generated**
- All 5 reels and YouTube video lack subtitles/captions
- Root cause: Subtitle generation requires `run_caption_adapter.py` from the VCE which was not runnable without render_manifest.json
- Fix needed: Add subtitle burn-in step (ffmpeg `drawtext` or external ASR tool)

**11. EPUB build not verified**
- `epub_build` stage shows `pass` in job.json, but no .epub file found in workspace
- Root cause: `scripts/release/build_epub.py` may write to a default path outside the workspace, or may be a no-op when called inline

## Pipeline Gaps Discovered

1. **No `deep_book_6h` format path** — Format key exists in catalog config but no code path expands prose to target word count.

2. **Podcast pipeline not book-aware** — `run_podcast_pipeline.py` pulls from weekly catalog queue, not from a specific just-generated book. There's no `--teacher` or `--book-id` override.

3. **CosyVoice2 API contract mismatch** — `generate_showcase_bundle.py` calls `/api/v1/tts` but the server exposes only the Gradio API. No production-ready REST wrapper for CosyVoice2 exists in the repo.

4. **No book → video render_manifest bridge** — Cannot go directly from `book.txt` to the VCE render_manifest format without the full catalog workflow.

5. **flux_client.py `_meta` node bug** — Blocks ComfyUI calls without workaround.

6. **Author vs teacher identity split** — Task spec used `--author adi_da` but should be `--teacher adi_da`. Docs don't clearly state teachers must use `--teacher` flag.

7. **No voice reference clips for Adi Da** — `assets/tts/reference_clips/` is empty. CosyVoice2 zero-shot clone cannot work without a reference WAV.

8. **Brand→teacher mapping inconsistent** — `brand_identity_system.yaml`, `generate_showcase_bundle.py`, and `generate_bestseller_covers.py` use different resolution paths for brand→teacher.
