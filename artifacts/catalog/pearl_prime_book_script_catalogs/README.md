# Pearl Prime Book Script Catalogs

Per-locale catalog/queue artifacts for the Pearl Prime book script pipeline.
**These are queue rows, not manuscripts.** Nothing in this directory was
generated by an LLM. Rows are deterministically synthesized from existing
registries; rendering happens later via the spine pipeline.

Generator: [`scripts/catalog/generate_pearl_prime_book_script_catalog.py`](../../../scripts/catalog/generate_pearl_prime_book_script_catalog.py)

## Pearl Prime structural lock

Every row stores the same Pearl Prime structural format. The catalog is
**format-locked** — nothing here negotiates structure with the renderer.

| Field                  | Value (literal)                                          |
|------------------------|----------------------------------------------------------|
| `section_plan_id`      | `pearl_prime_12x10x5`                                    |
| `variant_pool_size`    | `5`                                                      |
| `bestseller_overlay_ref` | `docs/PEARL_PRIME_BESTSELLER_WRITING_OVERLAY_SPEC.md`  |
| `selection_strategy`   | `deterministic_by_seed`                                  |
| `pipeline_route`       | `scripts/run_pipeline.py --pipeline-mode spine`          |
| `runtime_format`       | `standard_book` (default; from `config/format_selection/format_registry.yaml`) |

Pearl Prime structure (per [`docs/PEARL_PRIME_BOOK_SYSTEM_CANONICAL.md`](../../../docs/PEARL_PRIME_BOOK_SYSTEM_CANONICAL.md) §2):
**12 chapters × 10 sections × 5 variants per section** (1 of 5 selected at
render time by deterministic seed). Bestseller overlay is render-time
injection — the catalog stores only the spec reference.

## High-confidence filter (row inclusion logic)

Per owner decision Q4 (option **b**), this catalog is the high-confidence
subset, not the full Cartesian explosion.

For each `(brand, topic, persona)` triple in `(active_topics × active_personas) ∖ kill_list`:

1. Resolve `teacher_id` from the brand:
   - 12 teacher brands → from `config/catalog_planning/teacher_brand_archetypes.yaml`
   - `bright_presence_tw` → `adi_da` (Taiwan-only Adi Da teacher-mode brand)
   - 6 zh-specific standard brands per locale → from `brand_teacher_matrix_zh.yaml` (typically `ahjan`)
2. Look up `(teacher_id, topic)` and `(teacher_id, persona)` in `teacher_topic_persona_scores.yaml`.
3. `composite = (topic_score + persona_score) / 2` (per `composite_formula: average`).
4. Decision:
   - Both dimensions explicitly scored AND `composite ≥ 0.70` → emit, `readiness_status=ready`.
   - Either dimension absent (default 0.5 fallback) → emit, `readiness_status=blocked_score`, `blockers=needs_score`. The row is preserved per dev brief: *"If scoring data is missing for a row, mark it as needs_score rather than inventing confidence."*
   - Both explicit but `composite < 0.70` → drop (not high-confidence).

Kill-list filters applied at row-emit time:
- `kill_list.topic_persona_kills` (e.g. `compassion_fatigue × gen_alpha_students`)
- `kill_list.format_locale_kills`
- `kill_list.brand_locale_kills`

## Reproduction

```bash
python3 scripts/catalog/generate_pearl_prime_book_script_catalog.py \
  --locales en_US,ja_JP,zh_TW,zh_CN \
  --output-dir artifacts/catalog/pearl_prime_book_script_catalogs/
```

The generator is **deterministic** given identical input file contents.
`catalog_summary.json` records SHA-256 fingerprints of every config file
read so any drift is detectable.

## What this catalog does NOT do

- It does **not** generate manuscripts, chapters, sections, or section variants.
- It does **not** call any LLM (paid or free), per `CLAUDE.md` LLM Tier Policy.
- It does **not** cover manga catalogs — those are plan-only in this drop. See
  [`docs/MANGA_CATALOG_PLAN_EN_US_JA_JP_ZH_TW_ZH_CN_2026-04-28.md`](../../../docs/MANGA_CATALOG_PLAN_EN_US_JA_JP_ZH_TW_ZH_CN_2026-04-28.md).
- It does **not** validate atom existence or registry topic existence; that
  would gate `readiness_status=blocked_atoms` / `blocked_registry`. Today
  only scoring is gated. Atom-presence validation is a future iteration.
