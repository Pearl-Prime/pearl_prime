# Manga Catalogs — en_US + ja_JP

Per-locale manga catalog/queue artifacts. **These are catalog rows, not
panels or volumes.** Nothing in this directory was generated by ComfyUI,
no LoRA was trained, no LLM was called, no book/volume was assembled.

Generator: [`scripts/catalog/generate_manga_catalog.py`](../../../scripts/catalog/generate_manga_catalog.py)
Generated: 2026-04-28
Scope: en_US + ja_JP only. zh_TW + zh_CN catalogs deferred per current task brief.

## Source-of-truth chain

The catalogs are derived deterministically from these inputs:

1. **Allocation matrix** — [`config/manga/brand_genre_allocation.yaml`](../../../config/manga/brand_genre_allocation.yaml) (materialization of plan §2.1 / §2.2)
2. **Genre union + alias map** — [`config/manga/canonical_genre_list.yaml`](../../../config/manga/canonical_genre_list.yaml)
3. **Manga taxonomy** — [`config/manga/manga_taxonomy.yaml`](../../../config/manga/manga_taxonomy.yaml) (visual grammars + emotional/serialization engines)
4. **Pacing contracts** — [`config/manga/manga_pacing_by_genre.yaml`](../../../config/manga/manga_pacing_by_genre.yaml)
5. **Per-brand cadence** — [`config/manga/manga_brand_series_plan.yaml`](../../../config/manga/manga_brand_series_plan.yaml)
6. **LoRA plans** — [`config/manga/brand_lora_plans.yaml`](../../../config/manga/brand_lora_plans.yaml)
7. **Brand → teacher mapping** — [`config/catalog_planning/teacher_brand_archetypes.yaml`](../../../config/catalog_planning/teacher_brand_archetypes.yaml)
8. **Locale-native brand names** — [`config/catalog_planning/locale_brand_names.yaml`](../../../config/catalog_planning/locale_brand_names.yaml)

`manga_catalog_summary.json::source_files` records SHA-256 fingerprints of
all eight inputs so any drift is detectable.

## Series count rule

For each `(brand, genre)` cell with `allocation_pct > 0`:

```
series_count = max(1, round(allocation_pct / 10))
```

Examples (en_US `stillness_press`):

| Genre allocation | series_count |
|---|---|
| healing 40% (tentpole) | 4 series |
| essay 12% | 1 series |
| slice_of_life 10% | 1 series |
| memoir 8% | 1 series |
| supernatural_everyday 5% | 1 series |
| ... | ... |

Per-brand row counts cluster around 13–15 because each brand allocates
across ~12 non-zero genre cells whose percentages sum to 100%.

| 14 | `emotional_engine`      | Pipe-joined                                                        |

| 16 | `pacing_profile_ref`    | `genre_pacing.{key}` into `manga_pacing_by_genre.yaml`             |
| 17 | `lora_plan_ref`         | `brand_lora_plans.brand_style_loras.{brand}`                       |
| 18 | `character_pipeline_ref`| `brand_lora_plans.character_loras.{teacher}`                       |
| 19 | `pipeline_route`        | `MANGA_PIPELINE_TBD — see plan §7` (manga renderer is future work)  |
| 20 | `readiness_status`      | `ready` / `blocked_lora` / `blocked_pacing` / `blocked_archetype`  |

## Format-split bias per locale

Series format assignment is locale-biased per plan §4:

| Locale | print_manga | web_manga | webtoon_vertical |
|--------|------------:|----------:|-----------------:|
| en_US  |           0 |        40 |              130 |
| ja_JP  |          41 |        45 |               80 |

en_US has no `print_manga` rows because the lane content mix
(`catalog_generation_config.yaml::lane_content_mix.en_US`) is webtoon-primary
with KDP graphic-novel EPUB collected volumes (which the generator categorizes
as `web_manga`). ja_JP has `print_manga` for tentpole + ≥25% allocations and
`web_manga` for medium cells, reflecting Japan's dual track (tankōbon physical
+ LINE Manga digital).

## Tentpole reconciliation

Per plan §3 (option C — Coexist): the brand's **mono-genre profile**
(`config/source_of_truth/manga_profiles/brands/{brand}_{genre}.yaml`) is
the **tentpole / signature series**, while the matrix is the **portfolio
mix**.

Each row carries `is_tentpole=true|false`. Tentpole rows match the brand's
mono-genre filename. Total tentpole rows: 40 (en_US) + 39 (ja_JP).

### Tentpole mismatches (surface for owner)

When the matrix Primary cell does NOT match the mono-genre tentpole, the
row carries `tentpole_mismatch:matrix_primary={genre},tentpole={genre}` in
notes. Currently:

- **ja_JP `warrior_calm`** — matrix Primary = `battle 25%`, but tentpole =
  `cultivation` (per `warrior_calm_cultivation.yaml`). 2 series rows
  affected (`warrior_calm_battle_01`, `warrior_calm_battle_02`). Either
  re-author the mono-genre profile to `warrior_calm_battle.yaml` for
  ja_JP, or rebalance the matrix so cultivation ≥ battle. Owner decision.

## What this catalog does NOT do

- **No ComfyUI runs.** No images. No panels. No pages.
- **No LoRA training.** `lora_plan_ref` and `character_pipeline_ref` point
  to plan entries; training is a separate task on Pearl Star.
- **No volume assembly.** `chapters_per_volume` is a planning hint, not
  a manifest.
- **No LLM calls.** Series titles are deliberately blank with
  `needs_title_synthesis_locale_native` notes per the established Pearl
  Prime title-policy rule.
- **No zh_TW / zh_CN.** The matrix in `brand_genre_allocation.yaml`
  intentionally omits the Chinese locales. Adding them is the natural
  next step (the matrix is materialized for those locales in plan §2.3
  and §2.4).
